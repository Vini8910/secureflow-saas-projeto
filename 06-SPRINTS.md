# Processo de Sprint

Cada Sprint deve conter:

## Planejamento

-   Objetivo
-   Escopo
-   User Stories
-   Critérios de aceite

## Desenvolvimento

Informar:

-   Migrations
-   Models
-   Controllers
-   Services
-   Requests
-   Policies
-   Views
-   Rotas

## Qualidade

Executar:

-   Testes
-   Revisão
-   Validações

## Finalização

Atualizar:

-   CHANGELOG
-   Documentação
-   Status do Roadmap

Aguardar aprovação antes da próxima Sprint.
